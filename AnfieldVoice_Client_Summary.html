Security — Six Layers of Verification

No single point of failure. Every layer must be satisfied before anyone enters the estate.

1 Resident Authorisation

Every visit begins with an explicit resident action. No PIN, no entry. No exceptions.

2 Time-Bound PIN

Each PIN is valid for today only. Expired PINs are rejected automatically. Yesterday's PIN won't work today.

3 Ticket Retrieval

A PIN retrieves exactly one ticket, for one visitor, on one day. Revoked tickets are immediately denied.

4 Officer Verification

Security sees the full ticket and visually confirms identity. Every admission is attributed.

5 Exception Handling

If anything doesn't match, officer calls resident from the dashboard. No admission without resolution.

6 Complete Audit Trail

Every decision logged immutably — who created the ticket, when used, who admitted, and any exceptions.

How It Works — Visitor Scenarios

1. Standard Visitor

Resident creates ticket → System generates one-day PIN → Visitor enters PIN at gate → Security verifies and admits → All logged. ~20–30 sec

2. Recurring Visitor (Domestic Worker, School Transport)

Resident taps saved profile → "Generate Today's PIN" (5 sec) → Visitor enters PIN → Dashboard shows "Verified Profile" badge → Quick admit. ~10–15 sec No phone call needed.

3. Unexpected Visitor

Visitor has no PIN → Terminal calls resident via app → Resident answers and issues PIN → Standard admission. If resident doesn't answer, visitor is not admitted. Event is logged.

4. Delivery Management

Driver arrives → Resident pre-approval shown on dashboard (Accept / Hold at Gate / Redirect) → Delivery logged with driver details, time, and resolution. No phone calls.

Frequently Asked Questions

Q How long does installation take?
2–3 weeks from deposit to go-live. First week includes on-site support.
Q What happens if the internet goes down?
The on-site server runs independently — gate access and PIN verification continue even without internet. The dashboard and audit logs sync when connectivity resumes.
Q Can residents use their existing phones?
Yes. The resident app works on any iOS or Android phone. No special hardware needed for residents.
Q How are recurring visitors managed day-to-day?
Residents tap a saved profile and generate a fresh daily PIN in about 5 seconds. The security dashboard shows a "Verified Profile" badge for quick admit — no phone call needed.
Q What if a resident loses their phone?
The administration portal can reset access, issue temporary PINs, or transfer profiles. Residents can also log in from any device via the web portal.
Q Is my data safe and compliant?
All data is encrypted in transit and at rest. PINs are one-way hashed, never stored in plaintext. Fully POPIA-compliant with configurable retention policies.
Q What support is available after go-live?
SA-based support with local understanding. Remote monitoring, cloud backups, and phone support included. On-site visits available for complex issues.
Q Can the system handle multiple gates?
Yes. The platform supports multiple entry points with synchronised dashboards and centralised audit logging across all gates.
