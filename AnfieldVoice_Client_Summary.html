Security — Six Layers of Verification

No single point of failure. Every layer must be satisfied before anyone enters the estate.

1   Resident Authorisation

Every visit begins with an explicit resident action. No PIN, no entry. No exceptions.

2   Time-Bound PIN

Each PIN is valid for today only. Expired PINs are rejected automatically. Yesterday's PIN won't work today.

3   Ticket Retrieval

A PIN retrieves exactly one ticket, for one visitor, on one day. Revoked tickets are immediately denied.

4   Officer Verification

The security officer sees the full ticket and visually confirms the visitor matches. Every admission is attributed.

5   Exception Handling

If anything doesn't match, the officer calls the resident directly from the dashboard. No admission without resolution.

6   Complete Audit Trail

Every decision is logged immutably — who created the ticket, when it was used, who admitted the visitor, and any exceptions.

How It Works — Three Visitor Scenarios

1. Standard Visitor

Resident creates ticket → System generates one-day PIN → Visitor enters PIN at gate → Security verifies and admits → All logged. Total time: ~20-30 seconds.

2. Recurring Visitor (Daily — Domestic Worker, School Transport)

Resident taps saved profile → "Generate Today's PIN" (5 seconds) → Visitor enters PIN → Dashboard shows "Verified Profile" badge → Quick admit. Total time: ~10-15 seconds. No phone call needed.

3. Unexpected Visitor

Visitor has no PIN → Terminal calls resident via app → Resident answers and issues PIN → Standard admission. If resident doesn't answer, visitor is not admitted. Event is logged.

4. Delivery Management

Driver arrives → Resident pre-approval shown on dashboard (Accept / Hold at Gate / Redirect) → Delivery logged with driver details, time, and resolution. No phone calls.
